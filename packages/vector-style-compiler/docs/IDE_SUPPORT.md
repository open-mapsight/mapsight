# Mapsight Vector Style IDE Support

Mapsight vector styles are authored as a CSS/SCSS subset and compiled into
OpenLayers style functions. Standard CSS tooling already handles much of the
syntax, but Mapsight adds domain-specific selectors, properties, value helpers,
and runtime semantics that generic CSS inspections do not understand.

This maintainer note describes the IDE metadata that is currently generated and
published with `@mapsight/vector-style-compiler`. User-facing setup lives in the
package README.

## Goals

- Keep native CSS/SCSS editing support for syntax, imports, variables, and
  formatting.
- Add Mapsight-specific completion and hover docs for vector style properties,
  pseudo states, and helper functions.
- Reduce false positives from generic CSS inspections where custom metadata is
  enough.
- Keep the metadata generated from the same source data that documents supported
  Mapsight vector style properties.

## Current Support

The package generates static metadata for the two editor ecosystems we use most:

- VS Code custom CSS data: `mapsight-vector-style.css-data.json`
- JetBrains Web Types: `mapsight-vector-style.web-types.json`

Both files are generated by `scripts/update-ide-metadata.mts` from
`src/data/custom-css-properties.json` and written to `src/data/generated/`.
During `pnpm build`, the generated metadata is copied into `dist/` and published
with the package.

`package.json` exposes the metadata in two ways:

- `exports` entries for
  `@mapsight/vector-style-compiler/mapsight-vector-style.css-data.json` and
  `@mapsight/vector-style-compiler/mapsight-vector-style.web-types.json`
- a `web-types` field pointing JetBrains IDEs at
  `./dist/mapsight-vector-style.web-types.json`

## VS Code Setup

VS Code users can opt in through workspace settings:

```json
{
	"css.customData": [
		"./node_modules/@mapsight/vector-style-compiler/dist/mapsight-vector-style.css-data.json"
	],
	"scss.customData": [
		"./node_modules/@mapsight/vector-style-compiler/dist/mapsight-vector-style.css-data.json"
	]
}
```

The package export can also be used when the editor or tooling accepts package
subpaths:

```json
{
	"css.customData": [
		"./node_modules/@mapsight/vector-style-compiler/mapsight-vector-style.css-data.json"
	],
	"scss.customData": [
		"./node_modules/@mapsight/vector-style-compiler/mapsight-vector-style.css-data.json"
	]
}
```

The metadata contributes completions and hover text for Mapsight vector style
properties and state pseudo-classes.

## JetBrains Setup

JetBrains IDEs can discover the generated Web Types from the package
`web-types` field once `@mapsight/vector-style-compiler` is installed in the
project. The Web Types metadata contributes CSS property completions, hover
descriptions, state pseudo-classes, and Mapsight CSS function names.

The generated file includes:

- documented vector style properties from `custom-css-properties.json`
- case-preserving aliases such as `text-textAlign`
- Sass nesting helpers for property namespaces and nested segments
- `:selected` and `:highlighted` pseudo-classes
- `attr(...)`, `replace(...)`, and `mapsightRuntimeIcon(...)` function metadata

## Maintenance

Update `src/data/custom-css-properties.json` when adding or documenting vector
style properties, then run:

```sh
pnpm --filter @mapsight/vector-style-compiler generate:ide-metadata
```

The generator creates both editor metadata formats from the same property list.
`pnpm --filter @mapsight/vector-style-compiler build` also regenerates and
copies the files into `dist/`.

## Current Limits

Static metadata improves completion and hover docs, but it is not compiler-aware.
It cannot validate Mapsight-specific selector semantics, understand project
feature properties or environment keys, evaluate dynamic `attr(...)`/`calc(...)`
expressions, or report compiler-accurate source ranges.
